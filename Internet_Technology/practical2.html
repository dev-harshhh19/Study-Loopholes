Practical 2: Inter-LAN Connection

Aim

Create an inter-lan connection using routers.

Background Information

This practical demonstrates the concept of Inter-VLAN routing using a router in Cisco Packet Tracer. VLANs are logical networks created within a switch to separate broadcast domains. Since devices in different VLANs cannot communicate directly, a router is used to enable communication between them. This is achieved using the Router-on-a-Stick method, where a single router interface is divided into multiple sub-interfaces, each corresponding to a VLAN. Connectivity is verified using the ping command, and packet movement is visualized using CPT simulation mode.

Procedure

  1. Place the network devices

    Select one Router and place it on the workspace.

    Select one Switch (2960) and place it on the workspace.

    Select two PCs and place them on the workspace.

  2. Connect the devices

    Connect PC0 to Switch Fa0/1 using a copper straight-through cable.

    Connect PC1 to Switch Fa0/2 using a copper straight-through cable.

    Connect Switch Fa0/24 to Router Fa0/0 using a copper straight-through cable.

  3. Create VLANs on the Switch

    Click on the Switch and go to the CLI. Enter the following commands:

    enable 
configure terminal 
vlan 10 
name SALES 
exit 
vlan 20 
name HR 
exit

  4. Assign VLANs to Switch Ports

    Assign PC0 to VLAN 10:

    interface fastethernet 0/1 
switchport mode access 
switchport access vlan 10 
exit

    Assign PC1 to VLAN 20:

    interface fastethernet 0/2 
switchport mode access 
switchport access vlan 20 
exit

  5. Configure Trunk Port on the Switch

    Configure the port connecting the switch to the router as a trunk:

    interface fastethernet 0/24 
switchport mode trunk 
exit

  6. Configure the Router for Inter-VLAN Routing

    Click on the Router and go to the CLI.

    Enable the physical interface:

    enable 
configure terminal 
interface fastethernet 0/0 
no ip address 
no shutdown 
exit

    Create sub-interfaces for each VLAN.

    For VLAN 10:

    interface fastethernet 0/0.10 
encapsulation dot1Q 10 
ip address 192.168.10.1 255.255.255.0 
no shutdown 
exit

    For VLAN 20:

    interface fastethernet 0/0.20 
encapsulation dot1Q 20 
ip address 192.168.20.1 255.255.255.0 
no shutdown 
exit

  7. Configure IP Addresses on PCs

    PC0 (VLAN 10):
    IP Address: 192.168.10.2
    Subnet Mask: 255.255.255.0
    Default Gateway: 192.168.10.1
    PC1 (VLAN 20):
    IP Address: 192.168.20.2
    Subnet Mask: 255.255.255.0
    Default Gateway: 192.168.20.1

  8. Verify Connectivity Using Ping

    Go to PC0 ‚Üí Desktop ‚Üí Command Prompt and type:

    ping 192.168.20.2

    You will receive a reply, which shows that PC0 can communicate with PC1.

    Now go to PC1 ‚Üí Desktop ‚Üí Command Prompt and type:

    ping 192.168.10.2

    You will receive a reply, which confirms successful Inter-VLAN communication.

  9. Simulation Mode in Cisco Packet Tracer

    Switch to Simulation Mode in CPT.

    Click on Edit Filters and select ICMP and ARP.

    Run the ping command again from PC0 to PC1.

    Click Capture / Forward to observe packet movement from PC0 ‚Üí Switch ‚Üí Router ‚Üí Switch ‚Üí PC1 and back.
