Practical 2: Inter-LAN Connection

Aim

Create an inter-LAN connection between two separate networks using a router.

Background Information

This practical demonstrates how to connect two different LANs using a single router in Cisco Packet Tracer. Each LAN is connected to its own switch, and the router provides Layer-3 connectivity between the two networks. After assigning IP addresses to the PCs and configuring the router interfaces, connectivity is verified with ping and observed in Simulation Mode.

Procedure

  1. Place the network devices

    Select 1 Router (1941) and place it on the workspace.

    Select 2 Switches (2960) and place them on the workspace.

    Select 6 PCs and place them on the workspace.

  2. Connect the devices

    Connect PC0, PC1, PC2 to Switch0 using copper straight-through cables.

    Connect PC3, PC4, PC5 to Switch1 using copper straight-through cables.

    Connect Switch0 to Router Gig0/0 using a copper straight-through cable.

    Connect Switch1 to Router Gig0/1 using a copper straight-through cable.

  3. Configure IP addresses on PCs

    Assign static IP addresses to each PC.

    LAN 1 (Switch0 ‚Üí Router Gig0/0):

    PC0: 192.168.1.2 | Mask 255.255.255.0 | Gateway 192.168.1.1

    PC1: 192.168.1.3 | Mask 255.255.255.0 | Gateway 192.168.1.1

    PC2: 192.168.1.4 | Mask 255.255.255.0 | Gateway 192.168.1.1

    LAN 2 (Switch1 ‚Üí Router Gig0/1):

    PC3: 192.168.2.2 | Mask 255.255.255.0 | Gateway 192.168.2.1

    PC4: 192.168.2.3 | Mask 255.255.255.0 | Gateway 192.168.2.1

    PC5: 192.168.2.4 | Mask 255.255.255.0 | Gateway 192.168.2.1

  4. Configure the router

    Open the router CLI and configure both interfaces:

    enable
configure terminal

interface gigabitEthernet 0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit

interface gigabitEthernet 0/1
ip address 192.168.2.1 255.255.255.0
no shutdown
exit

end
write memory

  5. Verify connectivity using ping

    On any PC, open Desktop ‚Üí Command Prompt and test:

    ping 192.168.2.2

    If the configuration is correct, you should receive Reply from responses.

  6. Simulation Mode in Cisco Packet Tracer

    Switch to Simulation Mode, filter for ICMP and ARP, send a ping between LANs, then use Capture / Forward to observe packet flow: PC ‚Üí Switch ‚Üí Router ‚Üí other Switch ‚Üí destination PC.
