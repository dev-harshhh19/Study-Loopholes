Practical 7: DHCP Configuration

Aim

Simulate automatic IP address assignment using DHCP in Cisco Packet Tracer.

Background Information

Dynamic Host Configuration Protocol (DHCP) is used to automatically assign IP configuration parameters such as IP address, subnet mask, default gateway, and DNS server to hosts in a network. In this practical, a router acts as a DHCP server in a LAN environment.

Procedure

  1. Place Network Devices

    Place 2 PCs, 1 Switch (2960), and 1 Router (1941 or 2911) on the workspace.

  2. Connect the Devices

    Use Copper Straight-Through cables to connect PCs to the Switch and the Switch to the Router (GigabitEthernet0/0).

  3. Configure Router as DHCP in CLI

    Click Router0 ‚Üí CLI and enter the following commands:

    enable
configure terminal
interface gigabitEthernet 0/0
ip address 192.168.10.1 255.255.255.0
no shutdown
exit

ip dhcp excluded-address 192.168.10.1
ip dhcp pool LAN_POOL
network 192.168.10.0 255.255.255.0
default-router 192.168.10.1
dns-server 8.8.8.8
exit

  4. Configure PCs for DHCP

    For each PC: Desktop ‚Üí IP Configuration ‚Üí Select DHCP. The PCs will automatically receive IP addresses from the pool.

  5. Verify Assignment

    On PC Command Prompt, enter ipconfig to see the assigned details. On the Router, use show ip dhcp binding to see active leases.
