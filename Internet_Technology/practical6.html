Practical 6: Static IP (3 Routers)

Aim

Simulate static IP configuration using three routers, three switches, and six PCs in Cisco Packet Tracer.

Background Information

This practical demonstrates static IP configuration in a multi-network environment using three routers, three switches, and six PCs. Since routers do not support serial communication by default, HWIC-2T modules are installed to enable serial DCE/DTE connections. Each router connects different LANs, and static routes are configured to allow communication between all networks.

Procedure

  1. Place Network Devices

    Place 6 PCs, 3 Switches (2960), and 3 Routers (2911) on the workspace.

  2. Add Serial Modules (HWIC-2T)

    Perform the following for each router:

    1. Click router ‚Üí Physical tab.

    2. Turn OFF the power button.

    3. Drag HWIC-2T module into an empty slot.

    4. Turn ON the power.

  3. Connect the Devices

    PC to Switch: Copper Straight-Through cable.
    Switch to Router: Copper Straight-Through (Fa0/24 to Gig0/0).
    Router to Router: Serial DCE cable (S0/0/0 or S0/0/1).

  4. Configure IP Addresses

    Configure PCs with their respective LAN IPs and Gateways:

    • LAN 1: 192.168.1.0/24 (Gateway: 192.168.1.1)
    • LAN 2: 192.168.2.0/24 (Gateway: 192.168.2.1)
    • LAN 3: 192.168.3.0/24 (Gateway: 192.168.3.1)

  5. Configure Router Static Routes

    Enter the static routes in the CLI or Config tab:

    # Router0 
    
ip route 192.168.2.0 255.255.255.0 10.0.0.2
ip route 192.168.3.0 255.255.255.0 10.0.0.2
    # Router1 
    
ip route 192.168.1.0 255.255.255.0 10.0.0.1
ip route 192.168.3.0 255.255.255.0 10.0.0.6
    # Router2 
    
ip route 192.168.1.0 255.255.255.0 10.0.0.5
ip route 192.168.2.0 255.255.255.0 10.0.0.5

  6. Verify Connectivity

    Ping from PCs in different LANs (e.g., PC2 to PC5). Successful replies confirm correct configuration.
