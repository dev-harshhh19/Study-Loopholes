Practical 5: Hub as Repeater

Aim

Configure network connection using Hub cisco packet tracer

Background Information

A repeater is a physical layer (OSI Layer 1) device used to extend network distance. It receives a weak or distorted signal, regenerates it, and forwards it to all connected ports. In Cisco Packet Tracer, a Hub-PT is used to simulate a repeater since a hub is essentially a multi-port repeater.

Procedure

  1. Place End Devices and Hub

    Open Cisco Packet Tracer.

    From End Devices, place PC0 and PC1 on the workspace.

    From Network Devices ‚Üí Hubs, place a Hub-PT (Repeater).

  2. Connect the Devices

    Select Copper Straight-Through cable.

    Connect PC0 (FastEthernet0) to Hub (Port 1).

    Connect PC1 (FastEthernet0) to Hub (Port 2).

  3. Configure IP Addresses

    PC0 Configuration:
    Desktop ‚Üí IP Configuration
    IP Address: 192.168.1.1
    Subnet Mask: 255.255.255.0
    PC1 Configuration:
    Desktop ‚Üí IP Configuration
    IP Address: 192.168.1.2
    Subnet Mask: 255.255.255.0

  4. Verify Communication

    On PC0 ‚Üí Command Prompt, type:

    ping 192.168.1.2

    Observe successful replies.

    Repeat ping from PC1 to PC0.

  5. Simulation Mode

    Switch to Simulation Mode.

    Send a ping from PC0 to PC1.

    Observe that the signal is broadcast to all hub ports, demonstrating repeater behavior.
