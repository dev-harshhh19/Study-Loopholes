Practical 4: LAN with Switch

Aim

Configure network connection using switch in Cisco Packet Tracer.

Background Information

This practical demonstrates the basic concept of a Local Area Network (LAN) using a switch in Cisco Packet Tracer (CPT). Multiple end devices are connected through a switch and configured with IP addresses belonging to the same network. The switch forwards frames based on MAC addresses. Connectivity is verified using the ping command, which sends ICMP packets between devices. The simulation mode in CPT visually displays packet transmission, helping learners understand switching, addressing, and basic network troubleshooting concepts.

Procedure

  1. Place end devices and switch

    Select PC from the End Devices menu and place it on the workspace.

    Select another PC and place it on the workspace.

    Select a Switch (2960) from the Network Devices menu and place it on the workspace.

  2. Connect the Devices

    Use Copper Straight-Through cable.

    Connect PC0 (FastEthernet0) to Switch (FastEthernet0/1).

    Connect PC1 (FastEthernet0) to Switch (FastEthernet0/2).

  3. Configure IP Addresses

    PC0 Configuration:
    Desktop ‚Üí IP Configuration
    IP Address: 192.168.1.1
    Subnet Mask: 255.255.255.0 (auto-filled)
    PC1 Configuration:
    Desktop ‚Üí IP Configuration
    IP Address: 192.168.1.2
    Subnet Mask: 255.255.255.0 (auto-filled)

  4. Verify Connectivity

    On PC0: Desktop ‚Üí Command Prompt

    ping 192.168.1.2

    On PC1: Desktop ‚Üí Command Prompt

    ping 192.168.1.1

  5. Simulation Mode (Optional)

    Switch to Simulation Mode in Packet Tracer.

    Run ping from PC0 to PC1.

    Observe ICMP packets moving from PC0 ‚Üí Switch ‚Üí PC1.
