Practical 10: Web Server Configuration

Aim

Configure a Web Server in Cisco Packet Tracer.

Background Information

This practical demonstrates how to configure a web server to host websites and allow client PCs to access it using HTTP. A server device is configured with an IP address and web service enabled. PCs are connected via a switch and can access the web server using a browser.

Procedure

  1. Place Network Devices

    Place 2 PCs, 1 Switch (2960), and 1 Server (Server-PT-0) on the workspace.

  2. Connect the Devices

    Use Copper Straight-Through cables to connect the PCs and the Server to the Switch.

  3. Configure IP Addresses

    Server: 192.168.30.10
    PC0: 192.168.30.2
    PC1: 192.168.30.3
    Gateway: 192.168.30.1 (All devices)

  4. Enable Web Service

    1. Click Server ‚Üí Services ‚Üí HTTP.

    2. Turn the HTTP service ON.

    3. Edit index.html if you want to customize the home page.

  5. Test Web Server Access

    From PC0 or PC1: Desktop ‚Üí Web Browser. Enter URL: http://192.168.30.10.

  6. Simulation Mode (HTTP Request)

    Use Create Complex PDU on PC0. Set destination to server IP, application to HTTP, and observe the packet flow through the switch to the server and back.
