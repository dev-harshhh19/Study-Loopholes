Background Theory

ASP.NET Navigation Controls

Navigation controls in ASP.NET help users move between pages of a website in a structured and intuitive way. They improve usability by providing clear page hierarchy and easy access to different sections.

Navigation Controls Overview

Control Tag Description
SiteMapPath <asp:SiteMapPath> Displays breadcrumb navigation (Home > Products > Electronics) based on Web.sitemap
TreeView <asp:TreeView> Shows expandable hierarchical tree of nodes
Menu <asp:Menu> Horizontal or vertical navigation menu with optional sub-menus

SiteMapPath â€” How It Works

  • Reads the Web.sitemap XML file to know the page hierarchy
  • Automatically shows breadcrumb like: Home > Products > Electronics
  • The current page is shown as the last non-clickable item
  • No code-behind needed â€” it is purely declarative

TreeView â€” How It Works

  • Nodes are defined statically in ASPX or bound from a data source
  • Clicking a node triggers SelectedNodeChanged event
  • TreeView1.SelectedNode.Value gives the selected node's value

Menu â€” How It Works

  • Menu items are defined using <asp:MenuItem>
  • Sub-items appear as dropdowns on hover
  • Orientation="Horizontal" or "Vertical" controls layout

ASP.NET Master Pages

A Master Page defines a common layout (header, navigation, footer) that is shared across multiple pages of an application. Content pages "plug in" their unique content through ContentPlaceHolder regions.

Key Concepts

Concept Description
Master Page (.master) Contains the shared layout: header, nav, footer + ContentPlaceHolder
ContentPlaceHolder A region in the master page where each content page injects its unique content
Content Page (.aspx) Links to the master page via MasterPageFile="~/Site1.master"
<asp:Content> Tag in the content page that maps to a specific ContentPlaceHolderID

Benefits of Master Pages

  • Write layout HTML once, reuse across all pages
  • Changing header/footer in one file updates all pages
  • Enforces consistent branding and style across the site

Web.sitemap
xml 
<?xml version="1.0" encoding="utf-8" ?>
<siteMap xmlns="http://schemas.microsoft.com/AspNet/SiteMap-File-1.0">
  <!-- Root node: Products page -->
  <siteMapNode title="Products" url="~/Products.aspx">
    <!-- Child node: Electronics page (inside Products) -->
    <siteMapNode title="Electronics" url="~/Electronics.aspx" />
  </siteMapNode>
</siteMap>

        runat="server"
        OnSelectedNodeChanged="TreeView1_SelectedNodeChanged">

        <Nodes>
            <asp:TreeNode Text="Root Node" Value="Root">
                <asp:TreeNode Text="Child Node 1" Value="Child1" />
                <asp:TreeNode Text="Child Node 2" Value="Child2" />

WebForm1.aspx.cs
csharp 
using System;

namespace Prac5
{
    public partial class WebForm1 : System.Web.UI.Page
    {
        // Fires when user clicks a TreeView node
        protected void TreeView1_SelectedNodeChanged(object sender, EventArgs e)
        {
            // SelectedNode.Value gives the Value attribute of the clicked node
            string selectedNodeValue = TreeView1.SelectedNode.Value;
            Response.Write("You selected: " + selectedNodeValue);
        }
    }
}

Site1.master
aspx 
<%@ Master Language="C#" AutoEventWireup="true"
CodeBehind="Site1.master.cs"
Inherits="Prac5.Site1" %>

<!DOCTYPE html>
<html lang="en">
<head runat="server">
    <meta charset="UTF-8" />
    <title>Master Page Example</title>
</head>
<body>

<header>
    <h1>Welcome to My Website</h1>
    <nav>
        <ul>
            <li><a href="Home.aspx">Home</a></li>
            <li><a href="About.aspx">About</a></li>
        </ul>
    </nav>
</header>

<div class="content">
    <!-- ContentPlaceHolder: each content page fills this area with its own HTML -->
    <asp:ContentPlaceHolder ID="MainContent" runat="server" />
</div>

<footer>
    <p>&copy; 2026 My Website</p>
</footer>

</body>
</html>
